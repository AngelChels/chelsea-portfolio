Key Projects

Real work.
Measurable impact.

Three platform engineering initiatives that demonstrate how I approach complex infrastructure problems at enterprise scale.

Project 01

CI/CD Platform Migration & Optimisation

GitHub ActionsAWS EC2TerraformBash

The Problem

Nationwide's CI/CD platform relied on EC2 self-hosted runners — a setup that created persistent infrastructure costs, maintenance overhead, and scalability constraints that blocked engineering teams during high-demand release windows.

What I Did

Led the end-to-end migration from EC2-based runners to GitHub-hosted runners, redesigning the runner strategy across multiple pipelines. Audited existing workflows, identified optimisation points, rebuilt pipeline configurations for the new architecture, and validated performance and reliability across all environments before cutover.

Impact

Eliminated always-on EC2 infrastructure cost across the platform
Improved scalability — runners provision on demand, no bottlenecks
Reduced operational maintenance burden for the platform team
Increased pipeline reliability with managed GitHub-hosted environments

Project 02

Secure AWS Cloud Infrastructure with Terraform

TerraformAWSIAMS3CloudWatch

The Problem

Enterprise cloud environments require consistent, auditable, and repeatable provisioning. Manual infrastructure changes introduce drift, security risk, and compliance gaps — critical concerns in a regulated financial services organisation like Nationwide.

What I Did

Designed and deployed production-grade AWS infrastructure using Terraform — covering compute (EC2), storage (S3), access control (IAM), and monitoring (CloudWatch). Built modular, reusable Terraform configurations aligned with enterprise security standards, with remote state management ensuring infrastructure is tracked, version-controlled, and reproducible across teams.

Impact

Infrastructure fully version-controlled — zero manual drift
IAM policies enforce least-privilege access across all services
Modular Terraform codebase reused across teams and environments
Production-ready setup aligned with enterprise compliance posture

Project 03

Deployment Automation & Developer Experience

GitHub ActionsPythonBashPowerShell

The Problem

Engineering teams were spending significant time on manual deployment tasks and repetitive operational work — a pattern that slowed release cycles, introduced human error, and kept engineers away from higher-value product work.

What I Did

Automated deployment workflows end-to-end using GitHub Actions, Python, and Bash, replacing manual steps with reliable, auditable pipelines. Partnered directly with engineering teams to map friction points, then built automation that reduced release risk and improved deployment consistency. Contributed internal documentation and knowledge-sharing sessions to embed strong platform practices across the organisation.

Impact

Significantly reduced manual intervention across the release process
Deployment errors reduced through enforced pipeline gates
Engineering teams ship faster with less operational overhead
Team productivity improved through mentoring and knowledge transfer
